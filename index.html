Hermes Agent × Local AI

A complete, copy-pasteable guide to running Hermes Agent with a local MiniMax M3 model on your engineering firm's PC — with SSH access, GitHub, office tools, and automated backups.

↗ Architecture Overview

Everything runs on a single powerful PC. You access it from your laptop, phone, or another machine via SSH, Telegram, or a web dashboard.

🧠

MiniMax M3 (Local LLM)

428B parameter MoE model (23B active). Served via vLLM or Ollama on the PC. Provides reasoning, tool-calling, and code generation without any cloud API.

⚡

Hermes Agent

The self-improving agent that connects to the local model. Handles file editing, terminal commands, web browsing, delegation, and cross-session memory.

🔗

SSH + Gateway

SSH from any device on the network. Optional Telegram/Discord gateway for remote input. Hermes CLI works identically over SSH.

📁

File Access & Backups

Syncthing for live sync, Restic for encrypted versioned backups, and a shared network drive for office suite document access.

🐙

GitHub Integration

Hermes connects to GitHub via SSH deploy keys and the GitHub CLI. Can open PRs, run CI checks, create repos, and manage issues.

📋

Office Suite

OnlyOffice or LibreOffice for document editing. Shared directories across the team. Hermes generates .docx, .xlsx, .pptx files on demand.

1 Set Up the Local AI Phase 1/6

Install Ollama or vLLM and pull MiniMax M3 — the 428B MoE model that runs entirely on your hardware.

1.1 Install Ollama

bash
curl -fsSL https://ollama.com/install.sh | sh
ollama --version
curl http://localhost:11434/api/tags

1.2 Pull MiniMax M3

bash
ollama pull minimax-m3:q4_K_M
ollama ps
⚡ Alternative — vLLM For production workloads: pip install vllm and serve via python -m vllm.entrypoints.openai.api_server --model minimax/minimax-m3 --tensor-parallel-size 2 --port 8000

1.3 Create 64K Context Model

bash
cat > ~/Modelfile-m3-64k << 'EOF'
FROM minimax-m3:q4_K_M
PARAMETER num_ctx 64000
PARAMETER num_gpu 99
EOF

ollama create m3-64k -f ~/Modelfile-m3-64k

1.4 Verify API Works

bash
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "m3-64k",
    "messages": [{"role": "user", "content": "Say hello and identify yourself."}],
    "max_tokens": 100
  }'
✅ Expected JSON response with "MiniMax" in the content field.

2 Install & Configure Hermes Phase 2/6

Install the Hermes agent and point it at your local MiniMax model. Two configuration options below.

bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | sh
hermes --version

Option A: Interactive Setup

bash
hermes setup
# → Select "Custom Endpoint"
# → Enter: http://localhost:11434/v1
# → API key: no-key (or placeholder)
# → Model: m3-64k

Option B: Manual Config

Write to ~/.hermes/config.yaml:

yaml
model:
  default: "m3-64k"
  provider: "custom"
  base_url: "http://localhost:11434/v1"
  fallback:
    provider: "custom"
    base_url: "http://localhost:11434/v1"
    model: "llama3.2:3b"

Keep Model Loaded

Override Ollama's 5-minute idle timeout:

bash
sudo mkdir -p /etc/systemd/system/ollama.service.d
sudo tee /etc/systemd/system/ollama.service.d/override.conf << 'EOF'
[Service]
Environment="OLLAMA_KEEP_ALIVE=24h"
EOF
sudo systemctl daemon-reload
sudo systemctl restart ollama
✅ Done Run hermes to start a session — zero cloud calls.

3 SSH + Network Access Phase 3/6

Enable remote access so you can control Hermes from your laptop, phone, or another machine on the network.

bash
sudo apt install openssh-server
sudo systemctl enable --now ssh
sudo ufw allow from 192.168.1.0/24 to any port 22 proto tcp
# Adjust subnet to your LAN
🔒 Security Never expose SSH directly to the internet. Use Tailscale, WireGuard, or Cloudflare Tunnel. Disable password auth after copying your key.
bash
ssh-copy-id user@hermes-pc-ip
sudo sed -i 's/^#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
# or: s/^PasswordAuthentication yes/...
sudo systemctl restart ssh
✅ Verify ssh user@hermes-pc "ollama ps && hermes --version"

4 GitHub Integration Phase 4/6

Connect Hermes to GitHub so it can open PRs, run CI, create repos, and manage issues autonomously.

Install & Auth GitHub CLI

bash
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list
sudo apt update && sudo apt install gh
gh auth login --hostname github.com --git-protocol ssh

Add Hermes Helpers

Write to ~/.hermes/skills/github.md so Hermes understands your workflow:

markdown
# GitHub Workflow for Hermes
- Use feature branches: feature/<name>
- PR title format: "[type] Brief description"
- Always run CI before merging
- Squash-merge to main
- Delete branch after merge
✅ Ready Hermes can now create repos, open PRs, run CI checks, and manage issues.

6 Automated Backups Phase 6/6

Protect your Hermes config, models, skills, memory, and documents with live sync and encrypted versioned backups.

Live Sync (Syncthing)

bash
sudo apt install syncthing
sudo systemctl enable --now syncthing@$(whoami)
# Open http://localhost:8384 and add devices/folders

Versioned Backups (Restic + Backblaze B2)

bash
sudo apt install restic
restic init --repo b2:your-bucket-name:/
restic backup ~/.hermes ~/documents
# Or S3-compatible, SFTP, local disk
restic snapshots

Automate with Cron

bash
# Daily backup at 2 AM
crontab -e
# Add: 0 2 * * * restic backup ~/.hermes ~/documents --tag daily
✅ Protected Everything is synced live to your team's machines and versioned off-site daily.

! Troubleshooting fixes

Common issues and their solutions when setting up Local AI + Hermes.

▸ Ollama won't start / GPU not detected
Check: nvidia-smi must show your GPU. Install drivers: sudo apt install nvidia-driver-550 nvidia-utils-550 && sudo reboot. Then restart Ollama: sudo systemctl restart ollama. Check logs: journalctl -u ollama -n 50.

Container mode: If running Docker, add --gpus all and set NVIDIA_VISIBLE_DEVICES=all.
▸ Hermes can't connect to the model
Verify the model API is running: curl http://localhost:11434/api/tags. Check ~/.hermes/config.yaml for correct base_url and model. Try hermes setup again. Ensure Ollama isn't firewalled: sudo ufw status.
▸ Model is too slow / out of memory
Use a smaller quantized version (Q4_K_M or Q3_K_M). Reduce context: set num_ctx 16000. Close other GPU processes. For dual GPU: set tensor-parallel-size 2. Add more swap: sudo fallocate -l 32G /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile.
▸ SSH connection refused
Check SSH is running: sudo systemctl status ssh. Check firewall: sudo ufw status. Verify on another PC: ssh user@ip. Check /etc/ssh/sshd_config for Port and AllowUsers.
▸ Git / GitHub authentication fails
Regenerate SSH key: ssh-keygen -t ed25519 -C "your-email". Add to GitHub: gh auth login or manually in Settings → SSH keys. Test: ssh -T git@github.com.
▸ pip install fails (externally-managed-environment)
Ubuntu 24.04 blocks system pip. Use a virtual environment: python3 -m venv ~/venv && source ~/venv/bin/activate. Or install pipx and use pipx install. For system-wide: add export PIP_REQUIRE_VIRTUALENV=false (not recommended). Or use uv: pip install uv && uv pip install python-docx openpyxl python-pptx.
